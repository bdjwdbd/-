| ------- | ------------------ |

## 报告漏洞

如果您发现了安全漏洞，请**不要**通过公开的 Issue 追踪器报告。

### 报告流程

1. **发送邮件** 至 [在此处添加安全邮箱]
2. **邮件标题**: `[安全] 元灵系统漏洞报告`
3. **邮件内容**:
   - 漏洞描述
   - 影响范围
   - 复现步骤
   - 可能的修复方案（如有）

### 响应时间

- **确认收到**: 24 小时内
- **初步评估**: 3 个工作日内
- **修复时间**: 根据严重程度
  - 严重漏洞: 7 天内
  - 高危漏洞: 14 天内
  - 中危漏洞: 30 天内

### 披露政策

- 请勿公开披露漏洞，直到修复发布
- 我们会在修复后公开致谢（如果您愿意）
- 我们会在 CHANGELOG 中记录安全修复

## 安全最佳实践

### 使用建议

1. **保持更新**: 始终使用最新版本
2. **配置安全**: 
   - 启用熔断机制 (`enableCircuitBreaker: true`)
   - 启用限流机制 (`enableRateLimiter: true`)
   - 设置合理的超时时间 (`requestTimeoutMs`)
3. **权限控制**: 使用 RBAC 管理访问权限
4. **日志审计**: 启用审计日志，定期检查

### 敏感信息

- 不要在代码中硬编码密钥或密码
- 使用环境变量存储敏感配置
- 定期轮换 API 密钥和令牌

### 依赖安全

- 定期更新依赖包
- 使用 `npm audit` 检查已知漏洞
- 关注依赖包的安全公告

## 安全特性

元灵系统内置以下安全特性：

| 特性 | 说明 |
|------|------|
| **L4 灵盾层** | 输出验证、内容截断、权限控制 |
| **熔断机制** | 防止级联故障 |
| **限流机制** | 防止资源过载 |
| **沙盒隔离** | 四级沙盒隔离危险操作 |
| **风险评估** | 自动评估操作风险等级 |
| **追踪审计** | 全链路追踪和决策审计 |
