## [4.9.6] - 2026-04-16

### 新增

- **主流程优化**
  - 智能系统作为统一入口，带缓存和超时控制
  - L0 思考 + 记忆搜索并行执行，各带超时控制
  - 智能路由：根据意图置信度选择执行路径
  - Skill 实际执行：尝试调用 Skill 的 execute/handler/run 方法
  - 请求优先级管理：高/普通/低三级优先级
  - 请求取消机制：AbortController 支持

- **健壮性增强**
  - 超时控制：智能系统 30s、L0 思考 10s、记忆搜索 5s、Skill 执行 5s
  - 重试机制：默认最多重试 2 次，间隔 1 秒
  - 熔断机制：连续 5 次失败后熔断，30 秒后半开
  - 限流机制：每分钟最多 60 个请求（可配置）

- **性能优化**
  - 智能系统结果缓存（30 秒 TTL）
  - 请求去重（5 秒内返回缓存）
  - 性能监控采样（10% 采样率）
  - 模块预热完善（预热所有关键模块）

- **工程化**
  - 主流程单元测试 (`src/__tests__/main-flow.test.ts`)
  - 性能基准测试 (`src/benchmarks/main-flow.benchmark.ts`)
  - 版本检查脚本 (`scripts/check-version.js`)
  - 新增 npm 脚本：`test:main`、`test:all`、`benchmark:main`、`version:check`、`release`

### 修复

- 删除重复的 L4 验证调用
- 删除重复的性能记录
- 删除 `void sessionHistory` 冗余代码
- 修复 L0→L1 串行执行逻辑

### 优化

- 会话历史独立处理，失败不影响主流程
- 置信度阈值可配置（默认 0.8）
- 错误分类细化（8 种类型）
- 降级处理增强

  - 新增 CRITICAL 级别模式识别（rm -rf /、格式化等）

  - 调整 CRITICAL 阈值：0.75 → 0.8
