## 🏗️ 架构设计

### 1. 状态管理器（State Manager）

**核心原则**：LLM 无状态，外部托管所有复杂状态

```
┌─────────────────────────────────────────────────────────┐
│                    状态管理层                            │
│  ┌──────────────┬──────────────┬──────────────┐        │
│  │ SESSION      │ TASK         │ USER         │        │
│  │ 会话状态     │ 任务状态     │ 用户状态     │        │
│  │ (临时)       │ (中期)       │ (长期)       │        │
│  └──────────────┴──────────────┴──────────────┘        │
│  ┌──────────────┬──────────────┬──────────────┐        │
│  │ MEMORY       │ TOOL         │ SYSTEM       │        │
│  │ 记忆状态     │ 工具状态     │ 系统状态     │        │
│  │ (持久)       │ (临时)       │ (全局)       │        │
│  └──────────────┴──────────────┴──────────────┘        │
└─────────────────────────────────────────────────────────┘
```

**关键特性**：
- ✅ 6 种状态类别
- ✅ 检查点与恢复
- ✅ 审计日志
- ✅ 自动过期清理
- ✅ 分层存储（L1内存/L2文件）

### 2. 追踪系统（Trace System）

**核心能力**：全链路追踪，支持性能分析和异常检测

```
┌─────────────────────────────────────────────────────────┐
│                    追踪数据流                            │
│                                                         │
│  L6 灵识层 → L0 灵思层 → L1 灵枢层 → L2 灵脉层          │
│      ↓           ↓           ↓           ↓              │
│  [Span6]     [Span0]     [Span1]     [Span2]           │
│                                                         │
│  L3 灵躯层 → L4 灵盾层 → L5 灵韵层                      │
│      ↓           ↓           ↓                          │
│  [Span3]     [Span4]     [Span5]                       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**关键特性**：
- ✅ 全链路追踪（L0-L6）
- ✅ 跨度生命周期管理
- ✅ 决策审计记录
- ✅ 性能指标计算
- ✅ 异常检测
- ✅ 关键路径分析

### 3. Harness 主系统

**核心接口**：统一的状态管理和追踪接口

```typescript
// 状态管理
await harness.setState(key, value, category);
const state = await harness.getState(key);
const checkpointId = await harness.createCheckpoint(keys);
await harness.restoreCheckpoint(checkpointId);

// 追踪
const trace = harness.startTrace(name);
const span = harness.startSpan(operation, layer, trace);
harness.endSpan(span.spanId);
harness.endTrace(trace.traceId);

// PPAF 辅助
await harness.withTracing('operation', Layer.L1, async () => {
  // 自动追踪
});
await harness.withState('key', async () => {
  // 自动缓存
});
```

## 🎓 核心洞察

### 1. 无状态原则

> **LLM 本身不应该持有状态。所有复杂状态都应该由外部系统托管。**

这是可扩展性的基石。就像 HTTP 协议的无状态设计，让系统可以水平扩展。

### 2. 缰绳哲学

> **缰绳不是束缚，是安全释放 AI 全部潜能的保障。**

工程师的角色转变：从代码创作者升级为过程守护者。

### 3. 补偿面理论

> **每个 Harness 组件都编码了一条"模型做不到什么"的假设，当假设不再成立，组件就该走了。**

护城河不在 Harness 厚度，在迁移速度。
